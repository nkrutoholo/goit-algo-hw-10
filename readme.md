# Завдання 1. Жадібний алгоритм та динамічне програмування

Тут реалізовано два способи видачі решти монетами з набору [50, 25, 10, 5, 2, 1].

Перша функція find_coins_greedy працює жадібно, тобто бере найбільшу монету яка влізає, потім наступну і так далі. Друга функція find_min_coins використовує динамічне програмування і шукає саме мінімальну кількість монет. На суму 113 обидві повертають однакове: {50: 2, 10: 1, 2: 1, 1: 1}.

## Порівняння по часу

Заміряв час на різних сумах:

сума 100: жадібний 0.0004 ms, динамічний 0.0293 ms
сума 1000: жадібний 0.0003 ms, динамічний 0.2856 ms
сума 10000: жадібний 0.0003 ms, динамічний 2.5937 ms
сума 100000: жадібний 0.0002 ms, динамічний 28.9757 ms

## Висновки

Видно що жадібний працює майже за однаковий час, скільки б не була сума. Він просто пробігає по списку номіналів, а їх всього 6, тому сума на швидкість майже не впливає. Складність у нього виходить O(кількість номіналів).

Динамічне програмування навпаки залежить від суми. Коли сума росте в 10 разів, час теж приблизно в 10 разів більший. Це тому що алгоритм будує цілу таблицю розміром із суму і для кожного значення перебирає всі монети. Тобто складність O(сума на кількість монет). На великих сумах він стає реально повільним, на 100000 жадібний швидший в десятки тисяч разів.

Але є момент. Для нашого набору [50, 25, 10, 5, 2, 1] жадібний завжди дає правильний мінімум, бо набір зручний. А якби номінали були якісь дивні, наприклад [1, 3, 4], то жадібний міг би помилитись, а динамічне програмування все одно знайшло б оптимум.

Тому для звичайних грошей краще жадібний, він швидший і результат той самий. А динамічне брати коли набір монет довільний і нема впевненості що жадібний дасть мінімум.


# Завдання 2. Інтеграл методом Монте-Карло

Рахував площу під функцією f(x) = x в квадраті на відрізку від 0 до 2. Використав метод Монте-Карло, тобто метод середнього. Беремо багато випадкових точок x в межах від 0 до 2, рахуємо середнє значення функції в цих точках і множимо на ширину відрізка. Точок брав мільйон.

## Результати

Метод Монте-Карло дав 2.665137
Функція quad зі scipy дала 2.666667
Аналітично виходить 8/3, тобто теж приблизно 2.6667

## Висновки

Результат Монте-Карло майже співпав з точним значенням. Різниця всього близько 0.0015, тобто метод спрацював добре.

Сам метод не рахує точно, він наближає відповідь випадковими точками. Чим більше точок тим ближче до правильного значення. На мільйоні точок похибка вже зовсім маленька. Кожен раз число трохи інше, бо точки випадкові, але всі вони близько до правильного.

Перевірка через quad показала що рахунок вірний. Загалом Монте-Карло це простий спосіб приблизно порахувати інтеграл, особливо коли точну формулу знайти важко.
